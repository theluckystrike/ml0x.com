ML Algorithm Cheat Sheet — When to Use Each Algorithm

A decision guide covering 32 machine learning algorithms organized by category with use cases, strengths, weaknesses, and computational complexity.

By Michael Lip · Updated April 2026

Methodology

Algorithm selection criteria were compiled from scikit-learn documentation, Stanford CS229 course materials, and practical experience. Community relevance was validated via the StackExchange API v2.3 — "machine learning algorithm" questions collectively exceed 14K+ views on the top result, with 38 votes on predictive ordering. Training/prediction complexity follows standard references. Algorithms are grouped into 6 categories: Classification, Regression, Clustering, Dimensionality Reduction, Ensemble, and Deep Learning.

Decision Flowchart

Step 1: Labeled data? If yes, supervised (classification or regression). If no, unsupervised (clustering or dimensionality reduction).

Step 2: Output type? Categories = classification. Continuous values = regression. Groups = clustering. Compression = dimensionality reduction.

Step 3: Dataset size? Under 1K samples: Logistic Regression, SVM, KNN. 1K-100K: Random Forest, Gradient Boosting. Over 100K with unstructured data: Deep Learning.

Step 4: Interpretability? If required: Logistic Regression, Decision Trees, Linear models. If not: Gradient Boosting, Neural Networks.

The tabular data rule: For structured/tabular data, gradient boosting (XGBoost, LightGBM, CatBoost) almost always beats deep learning. Deep learning wins on images, text, audio, and video — domains where automatic feature learning replaces manual feature engineering.

Frequently Asked Questions

How do I choose the right machine learning algorithm?

Start with four questions: Is the problem supervised or unsupervised? Are you predicting a category or a number? How much labeled data do you have? Is interpretability required? With under 1K samples, try logistic regression, SVM, or KNN. With 1K-100K, try Random Forest or Gradient Boosting. With 100K+ and unstructured data, try deep learning. Always start simple and increase complexity only when needed.

When should I use Random Forest vs Gradient Boosting?

Random Forest is more robust to hyperparameter choices and harder to overfit — it works well out of the box. Gradient Boosting (XGBoost, LightGBM, CatBoost) typically achieves higher accuracy when properly tuned but requires more careful optimization. Use Random Forest for quick baselines, Gradient Boosting when you can invest in hyperparameter search.

What is the difference between classification and regression?

Classification predicts discrete categories (spam/not spam, cat/dog) while regression predicts continuous values (price, temperature). Many algorithms handle both: Decision Trees, Random Forests, SVMs, and neural networks all have classification and regression variants. The choice depends on your target variable type.

When is deep learning better than traditional ML?

Deep learning excels with large datasets (100K+ samples), unstructured data (images, text, audio), and sufficient compute. For tabular data under 100K rows, gradient boosting typically wins. Deep learning's advantage is automatic feature learning — it eliminates manual feature engineering but needs more data and compute to realize that benefit.

What are the most common clustering algorithms and when to use each?

K-Means for spherical clusters when you know K. DBSCAN for arbitrary shapes with automatic outlier detection. Hierarchical clustering for exploring multi-level structure. Gaussian Mixture Models for overlapping clusters with soft assignments. Spectral clustering for non-convex shapes in high-dimensional space.
