Machine Learning Formulas You Actually Need (With Examples)

Published April 2025 · 8 min read

Machine learning textbooks are filled with hundreds of equations, but the truth is that most practitioners use the same core set of formulas repeatedly. Whether you are training your first model or debugging a production pipeline, these are the formulas you need to understand deeply, not just memorize.

1. The Gradient Descent Update Rule

Every optimization algorithm builds on this foundation. The basic update is deceptively simple: adjust each parameter in the direction that reduces the loss, scaled by a learning rate.

theta = theta - learning_rate * gradient(loss, theta)

In practice, you rarely use vanilla gradient descent. Adam combines momentum (exponentially weighted moving average of gradients) with adaptive learning rates (exponentially weighted moving average of squared gradients). The bias-corrected estimates ensure the optimizer works well from the very first step.

A common mistake is setting the learning rate too high. If your loss spikes or oscillates wildly in the first few epochs, reduce it by a factor of 10. Most modern models train well with learning rates between 1e-4 and 3e-4 when using Adam or AdamW.

2. Cross-Entropy Loss

For classification, cross-entropy measures the distance between your predicted probability distribution and the true distribution. For binary classification:

L = -(y * log(p) + (1 - y) * log(1 - p))

Where y is the true label (0 or 1) and p is the predicted probability. For multi-class, this extends to summing over all classes. The key insight is that cross-entropy penalizes confident wrong predictions much more severely than uncertain ones. Predicting 0.99 for the wrong class produces a loss of about 4.6, while predicting 0.6 for the wrong class only produces 0.92.

3. Softmax Function

Softmax converts raw logits (any real numbers) into a valid probability distribution. Each output is between 0 and 1, and all outputs sum to 1:

softmax(z_i) = exp(z_i) / sum(exp(z_j) for all j)

The practical issue is numerical stability. If your logits are large (say, 1000), exp(1000) overflows. The standard fix is to subtract the maximum logit before computing: softmax(z - max(z)). Every major framework does this automatically, but it is worth understanding why when you encounter NaN values during training.

4. Attention Mechanism

The scaled dot-product attention formula is the heart of the transformer architecture that powers GPT, Claude, and every other large language model (compare models at gpt0x.com):

Attention(Q, K, V) = softmax(Q @ K.T / sqrt(d_k)) @ V

Q (queries), K (keys), and V (values) are linear projections of the input. The scaling factor sqrt(d_k) prevents the dot products from growing too large in high dimensions, which would push softmax into regions with tiny gradients. Multi-head attention runs this computation in parallel across multiple representation subspaces, then concatenates the results.

5. Batch Normalization

BatchNorm normalizes each feature to zero mean and unit variance within a mini-batch, then applies a learned affine transform:

y = gamma * (x - mean_batch) / sqrt(var_batch + epsilon) + beta

This solves the internal covariate shift problem: as earlier layers change during training, the distribution of inputs to later layers shifts, slowing convergence. BatchNorm stabilizes these distributions. The epsilon (typically 1e-5) prevents division by zero. Gamma and beta are learned parameters that allow the network to undo the normalization if needed.

6. Convolution Output Size

For any convolutional layer, you need to know the spatial dimensions of the output:

output_size = (input_size - kernel_size + 2 * padding) / stride + 1

This formula prevents the most common CNN debugging issue: dimension mismatches at the boundary between convolutional and fully connected layers. For a 32x32 input with a 3x3 kernel, stride 1, and padding 1, the output is 32x32 (same padding). With stride 2, it halves to 16x16.

7. R-squared (Coefficient of Determination)

For regression, R-squared tells you the proportion of variance in the target that your model explains:

R2 = 1 - sum((y_true - y_pred)**2) / sum((y_true - y_mean)**2)

An R-squared of 0.85 means your model explains 85% of the variance. A value of 0 means you are no better than predicting the mean. Negative values are possible when your model is worse than the mean predictor, which signals serious issues like data leakage in the test set or a fundamentally wrong model.

8. Precision, Recall, and F1

Classification accuracy is misleading for imbalanced datasets. Instead, use:

Precision = TP / (TP + FP)    # Of predicted positives, how many correct?
Recall    = TP / (TP + FN)    # Of actual positives, how many found?
F1        = 2 * P * R / (P + R)  # Harmonic mean of both

If you have 1000 transactions and 10 are fraud, a model that predicts "not fraud" for everything gets 99% accuracy but 0% recall. F1 score balances the trade-off. Use precision-weighted decisions when false positives are costly (spam filtering); use recall-weighted decisions when false negatives are costly (medical diagnosis).

Putting It Together

These eight formulas cover the core of what you need for building, training, and evaluating machine learning models. You can generate formatted cheat sheets for each algorithm on the ml0x.com homepage, complete with hyperparameter guides and code snippets.

The difference between a junior and senior ML engineer is not knowing more formulas; it is understanding fewer formulas more deeply. Know what each term controls, what happens when it goes wrong, and how to diagnose issues from the symptoms.
