Getting Started with Machine Learning from First Principles

A structured path from mathematical foundations to your first working ML implementation, with practical advice on avoiding the most common beginner mistakes.

Why First Principles?

The machine learning field moves fast. New papers appear daily, frameworks evolve quarterly, and the "best" approach to any given problem changes yearly. In this environment, chasing the latest library or tutorial creates a treadmill effect where you are always learning surface-level syntax without building lasting understanding. The alternative is to invest upfront in the fundamentals — linear algebra, calculus, probability, and optimization — and use that foundation to understand everything else as it appears.

When you truly understand how gradient descent works, you can read any optimizer paper and immediately grasp what it changes and why. When you understand the bias-variance tradeoff mathematically, you can diagnose model performance issues without guessing. This investment pays dividends across your entire career rather than expiring with the next framework release.

The Math You Actually Need

Many aspiring ML practitioners are intimidated by the mathematical prerequisites, but the reality is more approachable than it appears. You need three core areas, and you do not need to master them at a graduate level before writing your first line of ML code.

Linear algebra is the language of machine learning. Data points are vectors, transformations are matrices, and operations like projection and rotation underlie everything from PCA to neural network layers. Focus on vectors, dot products, matrix multiplication, eigenvalues, and the intuition behind matrix decomposition. Gilbert Strang's MIT OpenCourseWare lectures remain the gold standard.

Calculus is needed primarily for understanding optimization. Partial derivatives, the chain rule, and gradients are essential. You do not need to solve complex integrals or prove convergence theorems to be effective. The chain rule is the single most important concept because it is the mathematical basis of backpropagation — the algorithm that trains every neural network.

Probability and statistics provide the framework for thinking about uncertainty, which is inherent in all ML problems. Bayes' theorem, probability distributions (especially Gaussian), maximum likelihood estimation, and hypothesis testing give you the tools to reason about why models work and when they will fail.

Your First Implementation: Linear Regression

The best first algorithm to implement from scratch is linear regression with gradient descent. It touches every foundational concept: you represent data as matrices, define a loss function (mean squared error), compute its gradient analytically, and iteratively update parameters to minimize the loss. Here is a minimal implementation in Python using only NumPy: 

import numpy as np

# Generate synthetic data
np.random.seed(42)
X = 2 * np.random.rand(100, 1)
y = 4 + 3 * X + np.random.randn(100, 1) * 0.5

# Add bias term
X_b = np.c_[np.ones((100, 1)), X]

# Initialize parameters
theta = np.random.randn(2, 1)
learning_rate = 0.1
n_iterations = 1000

# Gradient descent
for i in range(n_iterations):
    gradients = (2 / len(X_b)) * X_b.T.dot(X_b.dot(theta) - y)
    theta -= learning_rate * gradients

print(f"Intercept: {theta[0][0]:.4f}, Slope: {theta[1][0]:.4f}")
# Expected: close to 4.0 and 3.0

This 15-line implementation contains the essence of all gradient-based optimization. The gradient computation, the learning rate, the iterative update loop — these same patterns appear in training a billion-parameter transformer. Once you can explain every line of this code and derive the gradient formula on paper, you have a foundation that will serve you for years.

Building Your Learning Path

After linear regression, follow this progression: logistic regression (to understand classification and the sigmoid function), decision trees (to see a non-gradient-based approach), k-nearest neighbors (to understand instance-based learning), and then a simple neural network with one hidden layer (to see how composition of simple functions creates powerful models). Implement each one from scratch before using library implementations.

For each algorithm, follow a three-step process. First, study the math: read the derivation, understand the loss function, and work through the gradient computation by hand. Second, implement it in NumPy with no external ML libraries. Third, compare your implementation against scikit-learn's version on the same dataset to verify correctness and understand what optimizations the library adds.

Common Mistakes to Avoid

The most common mistake is skipping the math and jumping straight to frameworks. This creates a fragile understanding that breaks down the moment you encounter a problem that does not fit a standard tutorial. The second most common mistake is trying to learn everything at once — pick one algorithm, understand it completely, and then move on.

Another frequent pitfall is neglecting data preprocessing. Feature scaling, handling missing values, and proper train-test splitting are not glamorous topics, but they determine whether your model works in practice. A perfectly implemented algorithm will fail on poorly prepared data.

Finally, resist the urge to jump to deep learning before understanding classical methods. Gradient boosted trees still outperform neural networks on most tabular data tasks, and understanding why requires familiarity with both approaches. Practitioners who explore model optimization techniques, such as those covered at Krzen, will find that classical foundations make advanced concepts far more accessible.

Next Steps

Once you have implemented the core algorithms from scratch, you are ready to graduate to using frameworks productively. At this point, PyTorch and scikit-learn become tools for speed rather than crutches that hide your lack of understanding. You will read documentation and immediately understand what each parameter controls. You will debug training issues by reasoning about gradients rather than blindly trying different hyperparameters. You will read research papers and see the equations as code you could write, not abstract symbols.

The first-principles path takes longer at the start but is faster in total. Every hour you invest in understanding the foundations saves many hours of confused debugging and tutorial-chasing later. Start with one algorithm, implement it completely, and build from there.
